今日主题：RAG

今天是周一，轮转回到 [[RAG]]。它仍然是当前 vault map 的 anchor topic：后续的 Agent、Evaluation、Acceleration、Online Learning 和 Mechanistic Interpretability 都会回到同一个问题，外部证据如何被选择、路由、压缩、消费、校验和保护。

今天不是 No Strong Signal。上周 RAG 主线已经从 retrieve more 转向 route better；今天的新信号进一步说明：原始 query 本身经常不够用。新的前沿正在尝试用未来任务模拟、隐藏态向量和受控检索实验来回答：该用什么 probe 去检索、什么时候证据已经够了、哪种检索策略真的支撑生成。

本周位置

Anchor topic。

周一固定回到 RAG，不是为了重复基础知识，而是为了给本周其他主题定参照系：Agent 需要检索路径，Evaluation 需要判断证据忠实度，Acceleration 需要降低上下文成本，Online Learning 需要决定哪些信息值得被内化。

本主题你现在需要知道的 3 件事

1. Query 不是检索意图的完整表达。

长期个性化记忆、多跳任务和工作流场景中，真正相关的证据可能和用户当前 query 语义相似度很低。

2. 检索充分性正在成为控制信号。

现在更关键的问题不是“多检一些”，而是判断何时已有证据足以生成，何时必须继续检索、换 probe 或换路线。

3. RAG 策略比较正在走向 controlled study。

只换一个检索模块、其他 pipeline 固定，并报告置信区间，比泛泛比较一堆 RAG 框架更有解释力。

What I Learned

  • Prospection-Guided Retrieval 是今天最值得先读的新论文：它指出标准 RAG 和 GraphRAG 仍然偏 retrospective，只按当前 query 或固定图遍历找过去；长期记忆检索需要先模拟未来可能步骤，再用这些步骤当 retrieval probes。
  • Latent Abstraction for RAG 把检索 query 从自然语言移到 LLM hidden state，并用 answer-token entropy 判断 retrieval sufficiency。这直接接上本地 RAG 主题里的“充分性 / 停止标准”张力。
  • 生物医学 RAG controlled study 提供了一个清醒的工程基线：固定 generation model、vector store 和 embedding 后，Cross-Encoder reranking 最稳，多查询扩展可能引入噪声；Dense baseline 与最优策略差距很小，不能过度解读单点分数。
  • 社区信号较弱但方向一致：r/Rag 近期讨论小模型 RAG workload、filter-first + RAG fallback 和 pipeline debugging，说明实际用户仍在围绕“先过滤、再检索、再修复”的工程路线打磨。

Key Patterns

1. 从 similarity retrieval 到 prospection-guided retrieval。

Thinking Ahead 提出 Prospection-Guided Retrieval：先把当前目标展开成未来可能步骤，再把这些步骤作为检索探针。它针对的是长期记忆里“重要事实和当前 query 不相似”的失效模式。

2. 从 text query 到 latent query。

Latent Abstraction for RAG 让 LLM 在隐藏态里直接产生 dense retrieval vector，并用 answer-token entropy 作为证据充分性信号。它把 retriever/generator 的边界变得更软。

3. 从 strategy leaderboard 到 controlled retrieval study。

Biomedical RAG controlled study 固定生成器、向量库和 embedding，只比较检索策略。结果提醒：Multi-Query Expansion 的 recall-oriented 直觉可能换来 contextual precision 下降。

4. 从 retriever routing 到 generation utility。

R^3AG 仍然是本地主线：检索器选择不应只看 retrieval quality，还要看 retrieved documents 是否真的帮助 generator 产出正确答案。

5. GraphRAG 的现实基准开始暴露细节损失。

WildGraphBench 显示 GraphRAG 对中等来源数量的 multi-fact aggregation 有帮助，但可能过度偏高层摘要，牺牲细粒度事实与 summarization 表现。

Read Now

1. Thinking Ahead: Prospection-Guided Retrieval of Memory with Language Models

  • why now： 这是 2026-05-13 的新提交，最能更新今天的 RAG 主线。
  • what changed： 它不再把 query 当成检索的全部，而是用 Tree-of-Thought 或 linear chain 模拟未来步骤，再用这些步骤检索长期记忆。
  • read it for： 理解长期个性化 RAG 为什么会被 query-reference low similarity 卡住。
  • how to start： 先看 MemoryQuest 的构造，再看 PGR-TOT 如何接近 3x recall，并检查 human annotations 是否支持 LLM judge preference。

2. Latent Abstraction for Retrieval-Augmented Generation

  • why now： 它直接回应 RAG 的“何时继续检索”问题，是检索充分性线的关键新入口。
  • what changed： LAnR 不生成自然语言查询，而是从 [PRED] token hidden states 生成检索向量，并用 answer-token entropy 判断证据是否足够。
  • read it for： 判断 retriever / generator 是否必须保持分离，以及 latent control head 能否替代显式多跳查询。
  • how to start： 先读 latent retrieval vector 和 sufficiency head，再看六个 QA benchmark 上 retrieval call reduction 与准确率的权衡。

3. Benchmarking Retrieval Strategies for Biomedical Retrieval-Augmented Generation: A Controlled Empirical Study

  • why now： 它不是最大的方法创新，但很适合作为工程基线，尤其在高风险 biomedical RAG 中。
  • what changed： 固定 GPT-4o-mini、ChromaDB 和 text-embedding-3-small，只比较 Dense、Hybrid、Cross-Encoder reranking、Multi-Query 和 MMR，并报告 95% confidence intervals。
  • read it for： 建立“检索策略差异是否真的显著”的判断习惯。
  • how to start： 先看 contextual precision / recall / faithfulness / answer relevancy 的组合，再看 Multi-Query 为什么可能引入噪声。

Watchlist

  • R^3AG: Retriever Routing for Retrieval-Augmented Generation

上周已经是主线，今天继续 watch。它把 retriever capability 拆成 retrieval quality 和 generation utility，适合和 PGR / LAnR 共同组成“query-aware control”读法。

  • WildGraphBench: Benchmarking GraphRAG with Wild-Source Corpora

GraphRAG 现实基准。它的价值是提醒：GraphRAG 能帮助多事实聚合，但不自动保留细粒度细节，尤其在 summarization 上可能弱。

  • MTRAG-UN: A Benchmark for Open Challenges in Multi-Turn RAG Conversations

多轮 RAG 仍值得跟。它把 unanswerable、underspecified、non-standalone questions 和 unclear responses 明确拉进 benchmark。

  • Case-Aware LLM-as-a-Judge Evaluation for Enterprise-Scale RAG Systems

适合连接 Evaluation / Judge Reliability。企业 RAG 不是只看答案是否相关，还要看 case identification、workflow alignment 和 partial resolution。

  • Search-P1: Path-Centric Reward Shaping for Stable and Efficient Agentic RAG Training

agentic RAG 训练线继续保留。它把 reward 从最终答案扩展到中间检索路径，是后续 deep research agent 和 RAG 的桥。
