今日主题：Acceleration / Long-Context Systems

今天是周四，轮转到 [[Acceleration]] / long-context systems。这条线对 vault map 的价值很直接：RAG、Agent、Deep Research 都越来越依赖长上下文，但真正的瓶颈不是“模型宣称窗口有多大”，而是 prefill / decode、KV cache、显存、吞吐和信息可达性是否支撑真实使用。

本轮最清楚的变化是：KV cache 正在从“需要压缩的内存负担”变成“可以被解释、递归利用、结构化管理的状态”。这比单纯比较压缩率更有价值，因为它开始回答一个更底层的问题：压缩之后，模型还能不能访问它以为自己保留下来的信息？

和 RAG 的关系

RAG 通常试图少塞上下文，只检索最相关证据；long-context systems 则试图让更多上下文可被模型直接处理。两者不是替代关系。今天的信号说明，即使你把证据塞进窗口，如果 KV cache eviction 或压缩破坏了 token-level routing，答案仍然可能看起来有依据、实际却访问不到关键证据。

本主题你现在需要知道的 3 件事

1. KV cache 不只是缓存，它正在变成递归状态。

KV-Fold 把 KV cache 当作 left fold accumulator（左折叠累积器）：每个 chunk 处理完后把新 KV 追加到已有 cache，再传给下一个 chunk。它的意义不是压缩，而是把 frozen transformer 里的缓存复用解释为一种无需训练的长上下文 recurrence。

2. 只看 attention weight 的 eviction 已经不够。

LaProx 指出，很多 KV eviction 方法只看局部 attention 权重，忽略 value states、output projection 和 inter-head interaction。它把 eviction 改写成 output-aware approximation（面向输出贡献的近似）问题。

3. 高压缩率可能隐藏安全悬崖。

Understanding the Physics of KV Cache Compression 区分 retention、accessibility 和 utilization：token 被保留下来，不代表语义仍可访问；benchmark 分数没掉，不代表内部路由没坏。

What I Learned

  • 05-12 的 KV-Fold 是今天最强的新读物。它没有提出新的压缩器，而是把 cache recurrence 做成 training-free protocol，并报告 16K 到 128K context 的 needle retrieval 在单张 40GB GPU 内保持 exact match。
  • 05-08 的 LaProx 把 KV eviction 从 head-wise attention averaging 推到 layer-wise output contribution。这个方向更接近真实推理路径，因为最终影响输出的是 attention map、value state 和输出投影的组合。
  • 本地已有的 Understanding the Physics... 是很好的诊断框架：它解释为什么“保留 KV 对”不等于“语义可达”，并提出 Global Eviction Ratio 与接近 90% 压缩时的 hallucination safety cliff。
  • 社区侧弱信号来自 LocalLLaMA：大家持续讨论 128K/262K context、VRAM、MTP 和吞吐。这不是论文证据，但说明端到端部署约束正在变成真实痛点。

Key Patterns

1. 从 eviction heuristic 到 cache recurrence。

KV-Fold 不问哪些 token 该删，而问 KV cache 能否作为跨 chunk 的稳定状态继续累积。它把 long-context inference 从“一次性长窗口”改写成“分块前向 + 状态传递”。

2. 从 token retention 到 semantic reachability。

Understanding the Physics... 的关键不是又提出一个压缩方法，而是提供诊断词汇：retention 是保留，accessibility 是还能不能被注意力路由到，utilization 是最终有没有被用于输出。

3. 从局部 attention 分数到输出贡献。

LaProx 的价值在于把 token importance 变成模型级可比较分数，而不是每个 head 局部决定谁重要。这对极端压缩尤其关键。

4. 从论文速度到硬件路径。

HillInfer 这类工作说明 KV 管理开始进入 CPU / SmartSSD / GPU 的异构内存调度问题。真正落地时，减少 cache 大小还不够，数据搬运和预取也会决定延迟。

Read Now

1. KV-Fold: One-Step KV-Cache Recurrence for Long-Context Inference

  • why now: 这是本轮最有新意的论文。它把 KV cache 看成跨 chunk 的状态累积器，而不是单次上下文里的被动缓存。
  • what changed: 方法不需要训练或改模型；每一步处理一个 chunk，把新 keys / values 追加到已有 cache，再传给下一步。作者报告 drift 短暂上升后进入稳定 plateau，并在 16K 到 128K needle-in-a-haystack 上保持 exact retrieval。
  • read it for: 判断 frozen transformer 是否已经隐含支持一种稳定的 KV-cache recurrence。
  • how to start: 先看 foldl 类比，再看 chain depth、chunk size 和 numerical precision 稳定性实验。注意它不是 bounded-memory streaming，cache 仍会增长。

2. Reformulating KV Cache Eviction Problem for Long-Context LLM Inference

  • why now: 这是当前 KV eviction 线最值得读的新方法之一。它正面批评只看 local attention weights 的做法太粗。
  • what changed: LaProx 把 eviction 改写成 layer-wise matrix multiplication approximation，显式纳入 projected value states、attention maps 和 inter-head dependencies。论文报告在 LongBench 和 Needle-In-A-Haystack 上只保留 5% KV cache 仍能维持性能。
  • read it for: 如果你关心极端压缩下为什么有些 token 不能按局部注意力分数删除，这篇提供了更系统的打分口径。
  • how to start: 先读它对 head-wise / weight-averaging 方法的批评，再看 model-wide token selection 如何生成全局可比 importance score。

3. Understanding the Physics of Key-Value Cache Compression for LLMs through Attention Dynamics

  • why now: 这篇本地已有 stub 和 transcript，是今天最适合补成精读的卡。它不是追求新 SOTA，而是解释压缩为什么会突然坏。
  • what changed: 它把 KV compression 解释为对 token-level routing 的扰动，并发现接近 90% 压缩时会出现 hallucination safety cliff；Global Eviction Ratio 上升与语义可达性崩塌相关。
  • read it for: 给所有 KV compression / eviction 论文加一个诊断框架：别只问 token 是否被保留，要问语义是否还可访问、是否真的被输出利用。
  • how to start: 先看 retention / accessibility / utilization 三分法，再看 multi-entity tracking、coreference、multi-hop reasoning 任务中的 failure pattern。
