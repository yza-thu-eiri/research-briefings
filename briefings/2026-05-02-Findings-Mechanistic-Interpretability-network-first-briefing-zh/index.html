Findings / Mechanistic Interpretability 晨间简报

今日主题

今天的轮转主题是 Findings / Mechanistic Interpretability。在当前 vault map 里，Findings 已经不再是“有趣论文大桶”，而是孵化器；真正成熟的子线应该退出并形成自己的主题面。Mechanistic Interpretability（机制可解释性）就是最成功的退出方向：它把模型行为从表面 benchmark 现象推进到 circuits（电路）、representations（表征）、FFN memory（前馈层记忆）、internal states（内部状态）这些更稳定的知识原子。

本周位置

这是 exploratory topic，但不是低价值探索。它离 RAG、Agent、Evaluation 都有距离，却决定我们能否理解模型内部“为什么会这样”。本地库当前把 Findings 维持为孵化层，而把 Mechanistic Interpretability 当成更长期的机制知识层。

和 RAG 的关系

RAG 主要解决“外部证据怎么进入模型输出”；机制可解释性追问的是“证据进入以后，模型内部如何绑定、存储、路由和使用它”。如果我们只看 RAG 的最终答案，很难知道失败来自检索、上下文绑定、事实存储、推理电路还是输出层偏差。机制可解释性可以为 RAG 失败诊断提供更深的内部层解释，尤其是实体绑定、事实回忆、上下文学习和隐藏状态表征这些环节。

本主题你现在需要知道的 3 件事

1. Findings 本体当前应保持收缩。 本地主题卡已经明确：Prompt Sensitivity 已退出，Hallucination Awareness 已分发到 Judge Reliability / Deep Research Agents，剩下的主要活跃孵化线是 Reasoning Length / Overthinking。

2. Mechanistic Interpretability 正在从事后解释转向可验证、可追踪、可部署。 最新工作不只是“找到一个有趣 head”，而是要求 circuit discovery 有 provable guarantees（可证明保证）、query-level explanation（单个输入的解释）和 training-data origin（训练数据来源）。

3. 机制证据的门槛在提高。 现在更值得读的是能建立因果、可复现或跨模型/跨样本稳定性的论文；单纯 probe 出一个方向、或者只给相关性热图，优先级应该下降。

What Changed

1. 自动电路发现开始要求形式化保证。 Formal Mechanistic Interpretability 是 ICLR 2026 Poster，把 automated circuit discovery 和 neural network verification 接起来，目标不是只找到“看起来合理”的 circuit，而是证明它在连续输入区域、patching 扰动和最小性约束下仍成立。

2. 机制解释从全局能力电路推进到单个查询电路。 Query Circuits 关注“为什么模型对这个具体输入给出这个输出”，而不是只解释 IOI 这类固定任务。它提出 Normalized Deviation Faithfulness，并报告 MMLU 单题上约 1.3% 的连接可恢复约 60% 性能。

3. 训练数据来源成为机制解释的新问题。 Mechanistic Data Attribution 用 influence functions 追踪可解释单元来自哪些训练样本，并通过删除 / 增强高影响样本因果调节 induction head 的形成。这直接连接本地经典锚点 In-context Learning and Induction Heads。

4. 解释工具本身开始被压力测试。 Pando 构造 model-organism benchmark，区分模型解释忠实、无解释、误导性解释三种情况。结果显示：当黑箱提示已能问出规则时，白盒工具不一定增益；当解释缺失或误导时，gradient-based attribution / RelP 才更有用。

5. 机制可解释性开始进入 reward model。 reward-lens 把 logit lens、activation patching、SAE 等工具迁移到只有标量 reward head 的 reward models，并发现 linear attribution 与 causal patching 在生产 reward model 上相关性很差。这对 RLHF / alignment 很关键。

Read Now

1. Formal Mechanistic Interpretability: Automated Circuit Discovery with Provable Guarantees

  • why now：这是今天最值得先读的 current-frontier paper。它把 circuit discovery 从启发式工具推进到带形式保证的算法问题，是机制可解释性成熟化的强信号。
  • method / benchmark / mechanism point：用 neural network verification 给三类保证：input domain robustness、robust patching、minimality；实验虽主要在 vision models 上，但方法论对 LLM circuit discovery 有方向性意义。
  • how to read it first：先读三类 guarantee 的定义，再看它们和传统 ACDC / patching 方法的差别；最后判断哪些保证能迁移到 Transformer LLM。
  • source link：https://openreview.net/forum?id=Timsb74vIY

2. Mechanistic Data Attribution: Tracing the Training Origins of Interpretable LLM Units

  • why now：它直接回答本地机制线的关键缺口：我们能找到 induction heads，但这些 circuit 是哪些训练数据塑造出来的？
  • method / benchmark / mechanism point：用 influence functions 追踪 interpretable units 的高影响训练样本；在 Pythia family 上通过删除或增强高影响样本，因果调节 induction head 形成和 ICL 能力。
  • how to read it first：先看 repetitive structural data（如 LaTeX / XML）如何成为 mechanistic catalyst；再看干预 induction head 时 ICL 能力如何同步变化。
  • source link：https://arxiv.org/abs/2601.21996

3. Pando: Do Interpretability Methods Work When Models Won't Explain Themselves?

  • why now：它是解释工具的压力测试。今天读它可以避免把 SAE、logit lens、circuit tracing 等方法默认当作可靠答案。
  • method / benchmark / mechanism point：构造 720 个带隐藏决策树规则的 finetuned model organisms，并控制模型是否给出忠实、缺失或误导性解释；比较黑箱 elicitation 和白盒工具。
  • how to read it first：先读 elicitation confounder（提示诱导混淆）是什么；再看为什么 RelP / gradient attribution 在误导解释条件下更有用，而 SAE / circuit tracing 不稳定。
  • source link：https://arxiv.org/abs/2604.11061

4. Query Circuits: Explaining How Language Models Answer User Prompts

  • why now：它把机制解释从“解释一个任务”推进到“解释一个具体 query”，这对 RAG/Agent 失败诊断更接近真实需求。
  • method / benchmark / mechanism point：提出 query circuit discovery，并用 Normalized Deviation Faithfulness 衡量稀疏 circuit 对单个输入决策的恢复能力；覆盖 IOI、arithmetic、MMLU、ARC。
  • how to read it first：先看 query circuit 与全局 capability circuit 的区别；再看 MMLU 单题上极稀疏连接恢复性能的例子。
  • source link：https://openreview.net/forum?id=wjrcQuDeSi

5. reward-lens: A Mechanistic Interpretability Library for Reward Models

  • why now：这是 2026-04-28 的新信号，说明机制可解释性正在进入 RLHF reward model，而不是只解释 generative LLM。
  • method / benchmark / mechanism point：把 reward head 的权重向量视为解释轴，提供 Reward Lens、component attribution、activation patching、SAE attribution 和 reward-hacking probe；关键负面发现是 linear attribution 不预测 causal patching effects。
  • how to read it first：先读为什么普通 logit lens 在 reward model 上失效；再看 observational attribution 和 causal patching 为什么必须并列比较。
  • source link：https://arxiv.org/abs/2604.26130

Watchlist

  • On the Optimal Reasoning Length for RL-Trained Language Models

对 Findings 当前唯一活跃孵化线很重要。它把 RL 训练后的长 CoT 输出拆成效率/性能权衡，指出过长会增加 dispersion，过短会 under-thinking。适合与 Between Underthinking and Overthinking 对读。

  • Do LLMs Overthink Basic Math Reasoning?

这篇更偏 empirical finding，但它提供 Overthinking Score，把准确率和 token efficiency 合成指标。适合判断 Reasoning Length 是否应该退出 Findings，进入 Test-Time Compute / Evaluation 交界线。

  • Causally Grounded Mechanistic Interpretability for LLMs with Faithful Natural-Language Explanations

关注 circuit-level attribution 到自然语言解释的桥接。值得 watch，但当前证据仍围绕 GPT-2 Small / IOI，范围比 Read Now 项窄。

  • MINAR: Mechanistic Interpretability for Neural Algorithmic Reasoning

把 attribution patching 迁移到 neural algorithmic reasoning / GNN 场景。适合关注“算法能力内部机制”的分支，和本地 States Hidden in Hidden States、Language Models Use Trigonometry to Do Addition 同向。
