今日主题：Agent / Deep Research Agents

今天是周二，轮转到 [[Agent]] / [[Deep Research Agents]]。在当前 vault map 里，这个主题不是泛泛追“agent 热度”，而是追一个更硬的问题：研究型 agent 能不能稳定完成“检索、证据覆盖、引用、归因、报告组织”这一整条链路。

它重要，是因为本库的 RAG 锚点已经从“检索到相关 chunk”扩展到“检索系统如何支撑长程研究任务”。Deep research agent 正好把 RAG 的召回、证据忠实度、停止条件和引用可靠性放到真实工作流里做压力测试。

本周位置

Adjacent topic。

RAG 是本周锚点；Agent / Deep Research Agents 是 RAG 的近邻扩展。它不是今天的 anchor topic，但它会反过来暴露 RAG 系统在真实研究流程里的瓶颈：检索是否足够全、证据是否可追踪、引用是否真实、agent 是否知道什么时候停止。

和 RAG 的关系

Deep research agent 可以看成“多轮、自主、带报告目标的 RAG”。普通 RAG 多半回答一个问题；deep research agent 要反复拆问题、发查询、读来源、更新证据状态、合并冲突信息、生成带引用的长报告。

所以今天最该关注的不是“agent 会不会调用工具”，而是四个 RAG 型问题：

  • 检索器能不能理解 agent 搜索前的推理意图。
  • 开放搜索空间里能不能兼顾召回和精度。
  • 引用是否真的支撑报告中的断言。
  • 错误发生在轨迹中的哪一步，而不只是最终答案错没错。

本主题你现在需要知道的 3 件事

1. Deep research agent 的前沿正在从“长报告生成”转向“过程可诊断”。 新论文不再只问报告像不像，而是问检索是否完整、引用是否有效、轨迹中哪一步开始出错。

2. 检索正在从人类短查询范式转向 agent reasoning-aware retrieval。 Agent 在搜索前通常已经写下任务阶段、证据缺口和搜索意图；如果检索器只看 query string，就浪费了最有价值的上下文。

3. 产业侧已经把 deep research agent 做成 API 产品面。 Google Gemini Deep Research Agent 文档显示，preview 版本已经强调 collaborative planning、visualization、MCP support，并明确提醒 web 与私有文件接入会引入 prompt injection、恶意网页和数据外泄风险。这说明 deep research 不再只是 benchmark 题，而是在进入可集成系统。

前沿变化观察

1. 评测对象从 answer quality 下沉到 evidence workflow。

DeepResearch Bench、DeepSearchQA、MMDeepResearch-Bench 共同形成一个趋势：研究型 agent 的能力不能只看最终报告，还要拆成证据覆盖、引用准确性、图文一致性、停止条件和组织结构。

2. “全面性”成为比单点命中更难的检索目标。

DeepSearchQA 的价值在于它不问 agent 能否找到一个答案，而是问它能否找全一组对象、去重、实体对齐，并判断何时停止。这非常贴近真实研究任务，也直接挑战当前 RAG 的 recall/precision 权衡。

3. 轨迹级可靠性正在成为 agent 评测的必要层。

AgentHallu 把“幻觉归因”定义为独立任务：不仅要知道输出错了，还要定位最早责任步骤。对 deep research agent 来说，这比最终答案审查更有修复价值。

4. 引用可靠性从人工审稿问题变成 agent workflow 问题。

BibAgent 把检索、推理、证据聚合和误引分类串起来，还用 Evidence Committee 处理付费墙下的间接证据。这条线和本库的 Judge Reliability、RAG Security 都有交叉。

5. 多 agent / 并行执行不只是效率叙事。

Kimi K2.5 的 Agent Swarm 和 Let It Flow 的 ALE/ROME 都说明，agent 工程正在转向轨迹生产、环境管理、后训练和上下文工程的系统化流水线。这里真正值得跟的是“并行后如何保持证据一致与责任可追踪”。

Read Now

1. DeepSearchQA: Bridging the Comprehensiveness Gap for Deep Research Agents

  • why now： 今天最该优先读。它直接击中 deep research agent 的核心短板：真实研究不是找到一个答案，而是系统性找全、去重、验证并停止。
  • method / benchmark / mechanism point： 900 条多步信息搜寻提示，覆盖 17 个领域；重点评测跨来源汇总、实体对齐、召回/精度平衡和开放搜索空间中的停止条件。
  • how to read it first： 先读任务定义和答案集合构造方式，再看失败模式。不要先看 leaderboard；先判断它如何把“全面性”变成可测量对象。
  • source link： https://arxiv.org/abs/2601.20975

2. AgentIR: Reasoning-Aware Retrieval for Deep Research Agents

  • why now： 它把 deep research agent 和 RAG 正面接起来：检索器不应只服务人类短 query，而要读懂 agent 当前的推理状态。
  • method / benchmark / mechanism point： 核心机制是把搜索前自然语言推理作为检索信号，利用其中的任务阶段、搜索意图和证据缺口。
  • how to read it first： 先看它如何表示“搜索前推理”，再看检索输入如何变化；最后对照本地 RAG Retrieval Recall 主题，判断哪些召回失败其实来自 query/context mismatch。
  • source link： https://arxiv.org/abs/2603.04384

3. AgentHallu: Benchmarking Automated Hallucination Attribution of LLM-based Agents

  • why now： 研究型 agent 的错误常沿轨迹传播，只查最终报告已经不够。你需要知道最早哪一步把系统带偏。
  • method / benchmark / mechanism point： 693 条高质量 agent 轨迹，覆盖 7 种 agent 框架和 5 个领域；提供 5 大类、14 个子类幻觉分类，并要求定位责任步骤。
  • how to read it first： 先读任务定义和标签体系，再看工具调用相关幻觉。目标是为后续 process monitor / citation checker 建立错误分类表。
  • source link： https://arxiv.org/abs/2601.06818

4. BibAgent: An Agentic Framework for Traceable Miscitation Detection in Scientific Literature

  • why now： Deep research agent 最容易“看起来专业”的地方就是引用；也最容易在引用处出问题。BibAgent 把这个点做成可追踪 workflow。
  • method / benchmark / mechanism point： 端到端误引检测框架，结合检索、推理、自适应证据聚合；提出 5 类误引体系和覆盖 254 个学科、6,350 个样本的 MisciteBench；Evidence Committee 用下游引用共识绕开部分付费墙问题。
  • how to read it first： 先读 MisciteBench 的误引分类，再读 Evidence Committee。把它和本地 GhostCite 线索连起来，而不是单独当 citation paper 看。
  • source link： https://arxiv.org/abs/2601.16993

5. Kimi K2.5: Visual Agentic Intelligence

  • why now： 如果只读 deep research benchmark，会漏掉 agent 系统工程的另一条强信号：多模态 + 并行 agent 编排正在成为模型报告的一等能力。
  • method / benchmark / mechanism point： Kimi K2.5 强调文本-视觉联合训练，并引入 Agent Swarm，把复杂任务动态拆分成异构子任务并并发执行；本地卡片记录其相对单 agent 基线最高降低 4.5 倍延迟。
  • how to read it first： 先读 Agent Swarm 架构，不要被大模型报告的综合 benchmark 淹没；重点问并行子任务之间的证据合并、冲突处理和责任追踪是否清楚。
  • source link： https://arxiv.org/abs/2602.02276

Watchlist

  • DeepResearch Bench: A Comprehensive Benchmark for Deep Research Agents

本地仍是 foundational entry，但今天不放 Read Now 第一位，因为它更像基座；若你还没有 deep research benchmark 地图，应把它作为背景读。

  • MMDeepResearch-Bench: A Benchmark for Multimodal Deep Research Agents

适合补多模态证据线。它的 FLAE / TRACE / MOSAIC 三轴评测很重要，尤其适合判断“图表证据是否真的支撑报告文字”。

  • Let It Flow: Agentic Crafting on Rock and Roll, Building the ROME Model within an Open Agentic Learning Ecosystem

放在工程化 watchlist。它不是单点方法，而是把环境、轨迹、后训练和上下文工程合成 ALE 生态。后续如果研究 coding/research agent 的训练流水线，需要回来读。

  • Gemini Deep Research Agent API documentation

产业信号强，但不是 paper。值得跟踪的是 MCP support、visualization、私有文件接入后的安全边界，以及官方文档中对 prompt injection、恶意网页和数据外泄的风险提示。
