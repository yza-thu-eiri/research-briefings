今日主题：Findings / Mechanistic Interpretability

今天是周六，轮转主题是 [[Findings]] / [[Mechanistic Interpretability]]。在当前 vault map 里，Findings 已经不是“所有有趣现象的大桶”，而是孵化器；真正有长期价值的线索应该退出并进入更稳定的机制主题。Mechanistic Interpretability 正是这个退出逻辑的核心承载面：它把模型行为从表面 benchmark 现象推进到 circuits、representations、FFN memory、internal states、training data origins 这些更可复用的知识原子。

今天的前沿信号不是“又发现一个有趣 head”，而是更像方法论升级：解释工作正在从人工找电路，转向 agent 自动提出机制假设、测试解释、发现特征；同时也出现了反向提醒：解释如果不能支持干预、调试或预测，就可能只是漂亮但不可行动的图。

本周位置

Exploratory topic。

它不是 RAG 锚点，也不是直接的系统工程线；但它决定我们能否解释 RAG / Agent / Evaluation 的底层失败机制。当前最适合把它当作长期知识原子层，而不是每天追热点的应用层。

和 RAG 的关系

RAG 关心外部证据如何进入模型输出；机制可解释性关心证据进入上下文后，模型内部如何绑定实体、路由注意力、调用事实知识、形成任务表征，以及何时在隐藏状态里已经知道但最终输出仍然失败。它不能替代 RAG 的工程评测，但能给 RAG 失败诊断提供更深一层解释：问题到底来自检索、上下文绑定、事实回忆、推理电路，还是输出层偏差。

本主题你现在需要知道的 3 件事

1. Findings 本体应继续收缩。 本地主题卡已经明确：Prompt Sensitivity 已退出，Hallucination Awareness 已分发，Reasoning Length / Overthinking 是当前仍留在 Findings 里的主要孵化线。

2. Mechanistic Interpretability 的证据门槛在提高。 当前更值得读的是能提供 causal intervention、formal guarantee、training-data attribution、query-level faithfulness 或 actionability 的论文；只给相关性热图或好看的 probe，优先级应下降。

3. 自动化解释正在成为新前沿。 2026 年的新线索开始让 agent 自动检查模型、提出假设、设计 probe、修正解释。这会改变 mechanistic interpretability 的工作方式，但也放大了“解释是否真的可验证”的问题。

前沿变化观察

1. Agentic interpretability 出现明确论文信号。

Automated Interpretability and Feature Discovery in Language Models with Agents 把解释过程做成 agent loop：从模型行为出发，自动形成机制假设、生成实验、解释特征。这比单次 SAE dashboard 更接近可迭代研究流程。

2. Steering 表征几何正在从“找到方向”推进到“半空间结构”。

CRH: Steering LLMs via Categorical Representation Halfspaces 把 category-level behavior steering 写成表示空间中的 halfspace 问题，强调类别级控制与 activation geometry 的联系。

3. 解释研究开始自我质疑 actionability。

Interpretability without Actionability 的价值在于提醒：解释不应只回答“我们能否描述模型内部”，还要回答“这个解释能否帮助干预、预测、调试或改进系统”。

4. Reward model 可解释性成为新迁移场景。

reward-lens 把 logit lens、activation patching、SAE 等工具迁移到只有标量 reward head 的 reward models，并发现 observational attribution 与 causal patching 的关系并不稳定。

5. Reasoning length 仍是 Findings 里最值得监控的孵化线。

本地队列里的 Between Underthinking and Overthinking、Do LLMs Overthink Basic Math Reasoning 和 On the Optimal Reasoning Length for RL-Trained Language Models 说明：推理长度不是简单“越长越好”，它正在和 test-time compute、RL 训练、评测成本连接。

Read Now

1. Automated Interpretability and Feature Discovery in Language Models with Agents

  • why now： 这是今天最有新意的方向信号。它把 mechanistic interpretability 从人工分析流程推进到 agentic workflow：让 agent 自动提出、测试、修正内部机制解释。
  • method / benchmark / mechanism point： 关键点不是单个 feature，而是自动解释循环：agent 读取模型行为与激活证据，生成机制假设，再通过实验反馈筛选解释。
  • how to read it first： 先读系统如何定义“feature discovery”和“hypothesis testing”，再看它是否有独立验证集或干预实验；重点判断 agent 生成解释是否只是语言总结，还是能产生可复现实验。
  • source link： https://arxiv.org/abs/2605.01555

2. CRH: Steering LLMs via Categorical Representation Halfspaces

  • why now： 它把 steering 从单一方向向量推进到 category-level halfspace（类别半空间）结构，适合连接本地“线性可分表征”“task vectors”“concept vectors”线索。
  • method / benchmark / mechanism point： 用表示空间中的半空间刻画类别行为，并用 activation steering 改变模型输出倾向；核心是看行为类别是否有稳定几何边界。
  • how to read it first： 先读 halfspace 定义和 category construction，再看 steering 是否跨 prompt、跨任务稳定。最后对照本地 Large Language Models Encode Semantics and Alignment in Linearly Separable Representations。
  • source link： https://arxiv.org/abs/2605.01844

3. Interpretability without Actionability: The Misalignment Between Understanding and Debugging in AI

  • why now： 这篇是必要的降温论文。它提醒我们：机制解释如果不能支持调试、干预、预测失败或改进模型，就可能只是理解幻觉。
  • method / benchmark / mechanism point： 核心不是提出新工具，而是重新界定 interpretability 的评估目标：从“是否看起来解释了模型”转向“是否能导出行动”。
  • how to read it first： 先读作者如何区分 understanding 和 debugging，再把它作为过滤器重读 SAE、circuit tracing、feature attribution 类论文。
  • source link： https://arxiv.org/abs/2603.18353

4. Mechanistic Data Attribution: Tracing the Training Origins of Interpretable LLM Units

  • why now： 它仍是本地机制线最该补的论文之一：我们不只想知道 induction head 存在，还想知道哪些训练数据塑造了它。
  • method / benchmark / mechanism point： 用 influence functions 追踪 interpretable units 的高影响训练样本，并通过删除 / 增强高影响样本因果调节 induction head 和 ICL 能力。
  • how to read it first： 先看 repetitive structural data 如何影响 induction head 形成，再读干预后 ICL 能力变化。它能把本地 In-context Learning and Induction Heads 与训练动态连接起来。
  • source link： https://arxiv.org/abs/2601.21996

5. reward-lens: A Mechanistic Interpretability Library for Reward Models

  • why now： RLHF / reward model 是 RAG 与 Agent 之外的关键评估和训练组件，但可解释性工具过去主要服务 generative LLM。reward-lens 把机制工具迁移到标量 reward head。
  • method / benchmark / mechanism point： 提供 Reward Lens、component attribution、activation patching、SAE attribution 和 reward-hacking probe；重要发现是 linear attribution 与 causal patching 在生产 reward model 上相关性很差。
  • how to read it first： 先看为什么普通 logit lens 不能直接用于 reward model，再看 observational attribution 与 causal patching 的差异。
  • source link： https://arxiv.org/abs/2604.26130

Watchlist

  • Pando: Do Interpretability Methods Work When Models Won't Explain Themselves?

解释工具压力测试。它用带隐藏规则的 model organisms 比较黑箱提示和白盒工具，适合回答“解释方法何时真有增益”。

  • Formal Mechanistic Interpretability: Automated Circuit Discovery with Provable Guarantees

上周 Read Now 仍应跟踪。它把 automated circuit discovery 和 neural network verification 接起来，是机制证据门槛提高的代表。

  • Query Circuits: Explaining How Language Models Answer User Prompts

关注单个 query 的机制解释，比全局任务电路更接近 RAG/Agent 失败诊断；适合后续和 query-level RAG attribution 连接。

  • On the Optimal Reasoning Length for RL-Trained Language Models

对 Findings 当前孵化线重要。它把 reasoning length 与 RL-trained model 的性能/成本曲线连接起来，可作为 Reasoning Length / Overthinking 是否退出 Findings 的判断依据。

  • Between Underthinking and Overthinking: An Empirical Study of Reasoning Length and Correctness in LLMs

本地队列项，适合作为 reasoning length 的基础入口。它不属于纯机制解释，但能帮助把“推理多长才够”从行为现象推进到 test-time compute / Evaluation 交叉问题。
