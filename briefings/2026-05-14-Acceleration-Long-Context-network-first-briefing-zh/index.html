今日主题：Acceleration / Long-Context Systems

今天是周四，轮转主题是 [[Acceleration]] / long-context systems。它在 vault map 里的位置很清楚：RAG、Agent、Deep Research 都想把更多材料放进上下文，但真正卡住系统的往往不是“窗口够不够大”，而是 prefill、decode、KV cache、GPU memory bandwidth 和多轮服务延迟能不能撑住。

今天不是 No Strong Signal。近两个月的论文信号很集中：KV cache 管理正在从“按 attention 分数删 token”的启发式，转向更可诊断的贡献估计、学习式预算、分层存储和系统级 SLO 管理。

本周位置

Adjacent topic。

本周锚点仍然是 RAG；Acceleration 是相邻基础设施主题。它不直接提升检索召回，但会决定 RAG 和 deep research agent 能否在长材料、长对话和反复工具调用中稳定运行。

和 RAG 的关系

RAG 和长上下文不是替代关系。RAG 负责缩小证据集合，长上下文负责容纳更完整的工作记忆；KV cache 和 context compression 负责让这件事在成本上可承受。今天的核心判断是：未来更像 RAG 先筛选，long-context system 再保真，KV/cache 策略按任务动态分配预算。

本主题你现在需要知道的 3 件事

1. KV eviction 正在从局部启发式变成贡献估计问题。

旧方法常把 attention weight 当成重要性；新论文开始把 value projection、output effect、head interaction 和 layer-wise contribution 放进判断。

2. 缓存不再只是“存或删”，而是多状态资源管理。

新路线会把 token 放进不同精度、不同层级、不同设备位置：full precision、quantized、evicted、HBM、DRAM、CPU-offloaded 都可能同时存在。

3. 系统论文开始主导真实部署问题。

只报告 memory reduction 不够了。现在要看 TTFT、TPOT、TBT、95th percentile latency、SLO attainment、CPU-GPU transfer 和 dynamic batching。

What I Learned

  • 2026 年 4-5 月新增信号的共同点是：KV cache 策略正在摆脱手写规则。 LKV 直接学习 head-wise budget 和 token selection；LaProx 把 eviction 改写成 output-aware matrix approximation。
  • TTKV、OrbitFlow、RetroInfer 说明 long-context acceleration 已经是存储系统问题，不只是模型算法问题。缓存的位置、精度、访问频率和跨设备搬运成本同样关键。
  • 本地队列里的 ATACompressor 仍然重要，但今天它更像“任务感知压缩”的桥：它告诉我们压缩预算应服务具体任务，而不是平均分给所有上下文。
  • 社区信号偏弱但方向一致：LocalLLaMA 近期讨论 Qwen 3.6 27B、128K-262K context、MTP、vLLM、Ollama 和消费级硬件吞吐，说明 acceleration 已经是实际用户痛点。

Key Patterns

1. 从 attention-only 到 output-aware。

Reformulating KV Cache Eviction Problem for Long-Context LLM Inference 指出只看局部 attention weight 会忽略 value representation、output projection 和 head interaction。它提出 LaProx，用 token 对最终输出的近似贡献来做全模型可比的重要性选择。

2. 从手写 budget 到 learned budget。

LKV 把 KV compression 设成端到端可微优化问题。它的重点不是又换一个 token scoring trick，而是证明 learned budgeting 是高压缩率下保持 fidelity 的主因。

3. 从单层缓存到 temporal tiers。

TTKV 把缓存按时间远近分成不同 tier，并在 HBM / DRAM 之间分配容量、精度和访问方式。这条线适合对照本地 EpiCache：一个偏时间层级，一个偏情节记忆。

4. 从算法压缩到 serving SLO。

OrbitFlow 和 RetroInfer 都把 KV cache 当成跨设备在线系统，核心目标是稳定吞吐和尾延迟，而不是单点 benchmark 分数。

5. 从事后逐出到写入前控制。

KV Admission 的问题定义很有用：不要等 cache 变大后再删，而是在 token 写入 KV cache 前预测它值不值得存。

Read Now

1. Reformulating KV Cache Eviction Problem for Long-Context LLM Inference

  • why now： 这是 2026-05-08 的新提交，最直接更新 KV eviction 的问题定义。
  • what changed： 它不再把 head-wise attention 平均当成重要性，而是显式建模 attention map 和 projected value state 的乘法交互。
  • read it for： 建立一个更严肃的 token contribution 视角，判断哪些缓存项真的影响输出。
  • how to start： 先读 LaProx 的 output-aware approximation，再看它如何做到 model-wide token selection。

2. LKV: End-to-End Learning of Head-wise Budgets and Token Selection for LLM KV Cache Eviction

  • why now： 它代表 learned KV eviction 线的新信号，2026-04-22 提交。
  • what changed： 论文把 budget allocation 和 token selection 都放进可学习框架，避免依赖 attention sink、recent window 或统计先验。
  • read it for： 判断高压缩率下，真正重要的是 token scoring 还是预算分配。
  • how to start： 先看 LKV-H 和 LKV-T 的分工，再看 LongBench / RULER 上 15% retention 的结果。

3. TTKV: Temporal-Tiered KV Cache for Long-Context LLM Inference

  • why now： 它把 KV cache 管理从“删 token”推进到“分层记忆系统”。
  • what changed： Temporal-Tiered KV Cache 把近期状态放在更快、更高精度 tier，把远期状态放到慢层，并用 block-wise streaming attention 重叠通信和计算。
  • read it for： 理解长上下文系统如何在 HBM / DRAM 之间交易 latency、precision 和 recall。
  • how to start： 先看 tier layout，再看 128K-context task 下 5.94x cross-tier traffic reduction 和 latency / throughput 报告。

4. KV Cache Optimization Strategies for Scalable and Efficient LLM Inference

  • why now： 如果今天只想补全地图，这篇综述最省时间。
  • what changed： 它把近期方法整理成 eviction、compression、hybrid memory、novel attention、combination strategies，并映射到 long-context single request、datacenter serving、edge device、multi-turn conversation、accuracy-critical reasoning 等场景。
  • read it for： 建立“没有单一最优 KV 策略”的判断框架。
  • how to start： 先读 taxonomy，再看它如何按 deployment scenario 推荐策略。

Watchlist

  • ParisKV: Fast and Drift-Robust KV-Cache Retrieval for Long-Context LLMs

适合跟踪 million-token context。它用 collision-based candidate selection 和 quantized reranking 做 on-demand top-k fetching，重点是 drift-robust retrieval 和 CPU-offloaded KV cache。

  • OrbitFlow: SLO-Aware Long-Context LLM Serving with Fine-Grained KV Cache Reconfiguration

VLDB 2026。它最值得关注的是 SLO-aware KV placement：根据 runtime feedback 动态决定每个 request、每层 cache 是否留在 GPU。

  • RetroInfer: A Vector Storage Engine for Scalable Long-Context LLM Inference

Microsoft Research / VLDB 线。它把 sparse KV cache 做成 vector storage engine，在 120K 到 1M token scale 上讨论 decoding throughput 和 retrieval cost。

  • ATACompressor: Adaptive Task-Aware Compression for Efficient Long-Context Processing in LLMs

本地高优先级队列项。它适合连接 RAG，因为它强调只压缩与当前任务相关的内容，并按相关内容长度动态调节压缩率。

  • Context Compression via Explicit Information Transmission

不是 KV eviction 本身，但很适合补 soft context compression。ComprExIT 用显式信息传输缓解逐层表征覆盖和压缩容量无协调分配。

  • KV Admission: Learning What to Write for Efficient Long-Context Inference

这篇适合作为 LKV / LaProx 的前置概念：缓存管理不只有 selection 和 eviction，还包括 admission，也就是“什么内容一开始就不该写入 cache”。
