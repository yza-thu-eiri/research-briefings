今日主题：Findings / Mechanistic Interpretability

今天是周六，轮转主题是 [[Findings]] / [[Mechanistic Interpretability]]。在当前 vault map 里，Findings 不是永久分类，而是孵化器；真正成熟的经验规律要么退出到更稳定主题，要么被降级。Mechanistic Interpretability 则是长期知识原子层：它追问模型内部到底如何存储事实、形成表征、执行电路、产生状态和推理。

今天不是 No Strong Signal。最强的新变化反而来自 Findings 里仅剩的活跃孵化线：Reasoning Length / Overthinking。2026-05-13 到 05-14 连续出现两篇论文，把“过度思考”从行为现象推进到两个更硬的问题：黑盒 DoS 攻击面，以及 RAG reranking 的推理成本控制。

本周位置

Exploratory topic。

它不是本周 RAG 锚点，也不是直接工程基础设施；但它决定我们能否把 RAG、Agent、Evaluation 的失败从表面现象追到内部机制、推理长度、状态表示和可干预结构。

和 RAG 的关系

RAG 关心证据是否被检索到、放进上下文、支持最终答案。机制可解释性关心的是：这些证据进入上下文后，模型内部是否正确绑定实体、激活相关事实、形成任务向量、避免被输出偏差覆盖。今天的 reasoning length 新信号还直接触到 RAG：listwise reranking 如果依赖长链式推理，可能在收益递减后继续烧 token。

本主题你现在需要知道的 3 件事

1. Findings 目前只剩一个主要孵化线：Reasoning Length / Overthinking。

Prompt Sensitivity 已经退出，Hallucination Awareness 已分发到 Evaluation / Judge Reliability 和 Deep Research Agents；今天的新论文说明 reasoning length 仍值得保留和跟踪。

2. 过度思考正在从效率问题变成安全与可用性问题。

如果攻击者能诱导 reasoning model 生成极长、冗余、反复自省的推理链，那么 overthinking 就是资源耗尽攻击面，而不只是“回答啰嗦”。

3. 机制可解释性的证据门槛继续提高。

上周的 agentic interpretability、Pando、reward-lens 和 mechanistic data attribution 共同指向同一要求：解释必须能被测试、被干预、与黑盒诱导区分，而不是只生成漂亮标签。

What I Learned

  • Inducing Overthink 是今天最强的新信号：它把 overthinking 设成黑盒攻击目标，通过结构化扰动诱导大型推理模型输出长达数十倍的推理链。这给 Reasoning Length / Overthinking 一个更明确的出口：它可能进入 Safety / Acceleration / Evaluation 的交叉区。
  • Stop Overthinking 把同一问题带到 RAG reranking：在 listwise reranking 中，更长 CoT 不总是带来更好排序；用长度正则化自蒸馏可以保留效果并减少 34%-37% 推理 token。
  • 机制可解释性侧没有完全改写上周判断，但更清晰了：自动化解释、受控可解释性 benchmark、reward model 可解释性和训练数据归因正在成为新的证据门槛。

Key Patterns

1. Overthinking 变成可攻击对象。

Inducing Overthink 用 hierarchical genetic algorithm 构造扰动，让黑盒 reasoning model 在不完整或矛盾输入上产生超长推理。它把推理长度从“效率指标”提升为 availability risk。

2. Reasoning length 开始接入 RAG 成本。

Stop Overthinking 研究 listwise reranking 中的 overthinking，提出 Length-Regularized Self-Distillation，在 TREC Deep Learning 和 NeuCLIR 上减少 34%-37% token，同时保持排序效果。

3. Agentic interpretability 仍是机制线的新工作流。

Automated Interpretability and Feature Discovery 用 agent loop 自动提出机制假设、生成 prompt controls、构建 activation-space kNN graph 并发现候选特征。关键要看它是否能产生可复现实验，而不是只自动写解释。

4. 解释方法开始被 controlled benchmark 反问。

Pando 控制模型解释是否 faithful，测试白盒 interpretability 是否真的超过黑盒 elicitation。结果提醒：并非所有白盒工具都有稳定增益，梯度 / RelP 目前更可靠。

5. Reward model 可解释性暴露 observational 与 causal gap。

reward-lens 把 logit lens、activation patching、SAE attribution 迁移到 reward models，并发现 linear attribution 与 causal patching 相关性很差。这对 RLHF / evaluator 系统很关键。

Read Now

1. Inducing Overthink: Hierarchical Genetic Algorithm-based DoS Attack on Black-Box Large Language Reasoning Models

  • why now： 这是 2026-05-13 的新提交，最直接更新 Reasoning Length / Overthinking 的问题定义。
  • what changed： 过度推理不再只是成本曲线，而是可被黑盒攻击诱导的资源耗尽风险。
  • read it for： 判断 reasoning model 在不完整、矛盾、缺失前提输入下是否存在系统性推理膨胀。
  • how to start： 先看 hierarchical genetic algorithm 如何构造扰动，再看 26.1x output length increase 的条件和 transferability。

2. Stop Overthinking: Unlocking Efficient Listwise Reranking with Minimal Reasoning

  • why now： 这是 2026-05-14 的新提交，直接把 overthinking 接到 RAG / retrieval reranking。
  • what changed： 论文证明 listwise reranking 中长 CoT 会出现收益递减，并用 self-distillation 选择“高质量但短”的 reasoning traces。
  • read it for： 理解 reasoning-enhanced reranker 何时该停止推理，以及如何把推理长度纳入排序系统预算。
  • how to start： 先看 Pareto-inspired filter，再看 token reduction 与 ranking effectiveness 是否同时保住。

3. Automated Interpretability and Feature Discovery in Language Models with Agents

  • why now： 它仍是机制可解释性近期最值得读的新方向：把解释流程变成 agentic empirical loop。
  • what changed： 从 one-shot auto-interpretation 转向假设提出、prompt control、multi-metric evaluation 和 feature discovery 的循环。
  • read it for： 判断自动化解释能否产出更 falsifiable 的机制假设。
  • how to start： 先看 explanation refinement loop，再看 feature discovery 的 activation-space graph 和 semantic coherence 标准。

Watchlist

  • Pando: Do Interpretability Methods Work When Models Won't Explain Themselves?

适合用来校准解释工具价值。它控制模型解释渠道是否 faithful，结果显示当解释缺失或误导时，gradient / RelP 有 3-5 个百分点增益，而 SAE、logit lens、circuit tracing 未稳定优于黑盒基线。

  • reward-lens: A Mechanistic Interpretability Library for Reward Models

适合和 Evaluation / RLHF 线连接。它把机制可解释性工具迁移到 scalar reward head，并把 observational attribution 与 causal patching 的分歧作为一等信号。

  • Mechanistic Data Attribution: Tracing the Training Origins of Interpretable LLM Units

仍是本地机制线的长期高价值入口。它用 influence functions 追踪 induction head 等 interpretable units 的训练数据来源，并通过删/增高影响样本做因果验证。

  • Causally Grounded Mechanistic Interpretability for LLMs with Faithful Natural-Language Explanations

适合补“机制解释如何变成自然语言解释”的证据链。它把 activation patching 找到的 attention heads 与 ERASER-style faithfulness metrics 接起来。

  • On the Optimal Reasoning Length for RL-Trained Language Models

reasoning length 的背景线。它说明 length penalty 可能阻碍推理能力获得，但适当 length control 可改善强先验 reasoning model 的效率。
