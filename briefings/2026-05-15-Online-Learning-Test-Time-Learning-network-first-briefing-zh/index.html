今日主题：Online Learning / Test-Time Learning

今天是周五，轮转到 [[Online Learning - Test-Time Learning]] / adaptation。这条线对 vault map 很重要，因为它追问的是一个比 agent 编排更底层的问题：模型上线后，能不能继续学习；如果能，学习信号来自哪里，更新写到哪里，又怎样避免把原有能力冲掉？

今天不是 No Strong Signal。过去一周最强的新信号是 2026-05-12 的 Learning, Fast and Slow 和 2026-05-06 的 Attribution-Guided Continual Learning。它们把主题从“测试时能不能更新”推进到更细的问题：哪些信息该放进上下文式 fast weights，哪些该写入参数式 slow weights，哪些参数在新任务中应该保持稳定。

本周位置

Exploratory-to-adjacent topic。

它还不是本周锚点，但已经不只是旁支。RAG 解决“去哪找证据”，Acceleration 解决“怎么承载长上下文”，而 Online Learning / TTT 解决“模型是否把新证据真正内化”。

和 RAG 的关系

RAG 默认模型本体不变，只把新知识放在外部检索和上下文里。Test-time learning 则更激进：它允许模型在部署时用当前输入、反馈、合成样本或 verifier 信号更新自身。两者未来可能分工：RAG 负责可追溯证据与外部记忆，TTT 负责把反复出现的局部规律压进 fast weights 或轻量参数更新。

本主题你现在需要知道的 3 件事

1. TTT 不等于普通 test-time compute。

test-time compute 通常是多想几步、多采样或多验证；TTT 会真的更新参数、fast weights 或内部表征。

2. 核心风险是灾难性遗忘。

如果部署期每次都把新任务写进参数，模型可能快速丢掉旧能力。近期论文开始用 attribution、gradient similarity 和 fast/slow split 来控制漂移。

3. 验证器正在变成在线学习的闸门。

自生成数据只有在被 verifier、teacher 或失败轨迹分析过滤后，才适合作为测试时训练信号；否则自我改进很容易变成自我污染。

What I Learned

  • 这条线正在从“让模型在测试时更新”转向“把更新写到正确的位置”。Learning, Fast and Slow 把优化后的上下文视为 fast weights，把模型参数视为 slow weights，试图避免所有任务信息都进入参数。
  • In-Place TTT 代表架构侧推进：它把标准 Transformer MLP 的 final projection matrix 当作 fast weights，并把测试时目标对齐到 next-token prediction，让 TTT 更像 drop-in 能力。
  • Attribution-Guided Continual Learning 代表稳定性侧推进：它不问“要不要继续训练”，而问每一层、每个参数对旧任务和新任务分别有多重要。
  • 本地队列里的 Test-Time Meta-Adaptation with Self-Synthesis、Continuous Self-Improvement ... Verifier-Driven Sample Selection 仍然是关键桥梁：它们共同说明测试时学习的瓶颈不只是更新算法，也是训练样本质量。

Key Patterns

1. 从 single-speed learning 到 fast / slow split。

Learning, Fast and Slow 提出 Fast-Slow Training：用可优化上下文承接快速任务适应，用模型参数保留较慢、较稳定的能力。这比“只更新参数”更接近真实部署需求。

2. 从外接 TTT 模块到 in-place fast weights。

In-Place Test-Time Training 不要求从头训练特殊 TTT 架构，而是把已有 LLM 的 MLP final projection matrix 作为可更新 fast weights，并用 chunk-wise update 支持长上下文。

3. 从粗粒度冻结到 attribution-guided update。

Attribution-Guided Continual Learning 用 task-specific element-wise parameter importance 调节梯度。它的隐含判断是：稳定性不是“少训练”，而是知道哪些参数不能乱动。

4. 从自生成样本到可控自生成课程。

Test-Time Meta-Adaptation with Self-Synthesis 和 TTSR 都在解决同一个难题：测试问题太难时，直接用模型答案当伪标签不可靠；更好的做法是合成针对性样本或由 teacher 角色分析失败轨迹。

5. 从现象级 forgetting 到机制级 forgetting。

2026 年 1 月的两篇机制论文把灾难性遗忘拆成 gradient interference、representational drift、loss-landscape flattening 或 gradient similarity，这给后续在线更新边界提供了可诊断指标。

Read Now

1. Learning, Fast and Slow: Towards LLMs That Adapt Continually

  • why now： 这是 2026-05-12 的新提交，也是今天最值得先读的论文，因为它直接给出 fast / slow learning 的系统框架。
  • what changed： 它不把学习限制在 in-context 或 in-weights 二选一，而是把 optimized context 当作 fast weights，让 slow weights 少漂移。
  • read it for： 判断在线学习能否在减少 catastrophic forgetting 的同时保留可塑性。
  • how to start： 先读 fast weights / slow weights 的定义，再看 FST 相比 parameter-only RL 的 sample efficiency、KL divergence 和后续任务适应结果。

2. Attribution-Guided Continual Learning for Large Language Models

  • why now： 这是 2026-05-06 的新提交，正好补上 TTT / continual adaptation 最大的安全缺口：哪些参数该更新，哪些该保护。
  • what changed： 它用 task-specific、element-wise 的参数重要性来调节梯度，而不是只靠 replay、regularization 或整块冻结。
  • read it for： 建立“语义感知的参数可塑性”视角，判断在线学习如何避免遗忘旧任务。
  • how to start： 先看 attribution score 如何估计，再看它怎样在每层 modulate gradients。

3. In-Place Test-Time Training

  • why now： 这是当前本地队列最核心的架构论文之一，也已经有 ICLR 2026 会议论文信号。
  • what changed： 它把 TTT 做成标准 Transformer 的 drop-in enhancement：更新 MLP final projection matrix，并把 objective 对齐到 autoregressive next-token prediction。
  • read it for： 理解“部署期继续学习”怎样从概念变成可插入 LLM 的机制。
  • how to start： 先看 fast weights 选在 MLP projection 的理由，再看 128K context 和 context parallelism 结果。

Watchlist

  • Test-Time Meta-Adaptation with Self-Synthesis

MASS 让模型在测试时为当前问题合成训练样本，再做 targeted self-update。适合跟 verifier-driven self-improvement 一起读，重点是合成样本是否真的能减少错误传播。

  • TTSR: Test-Time Self-Reflection for Continual Reasoning Improvement

用同一个模型在测试时扮演 Student 和 Teacher：Student 解题，Teacher 分析失败轨迹并合成变体题。它更偏 reasoning improvement，不一定是通用在线学习，但对“失败驱动学习”很有启发。

  • Continuous Self-Improvement of Large Language Models by Test-time Training with Verifier-Driven Sample Selection

本地高优先级队列项。它把 verifier 当作测试时训练样本的闸门，适合作为所有 self-generated TTT 方法的安全基线。

  • Continual Learning in Large Language Models: Methods, Challenges, and Opportunities

综述入口。它不一定最前沿，但能帮你把 continual pre-training、continual fine-tuning 和 continual alignment 放在一张图里。

  • Mechanistic Analysis of Catastrophic Forgetting in Large Language Models During Continual Fine-tuning

适合作为稳定性背景。它把 forgetting 关联到 attention weight 的梯度干扰、中间层表征漂移和 loss landscape 变化。

  • Learning the Mechanism of Catastrophic Forgetting: A Perspective from Gradient Similarity

从 gradient similarity 解释 forgetting，并区分 conflicting neurons 与 collaborative neurons。可与 attribution-guided continual learning 对照。
