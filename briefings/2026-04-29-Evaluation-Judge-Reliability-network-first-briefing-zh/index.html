Evaluation / Judge Reliability 晨间简报

今日主题

今天的轮转主题是 Evaluation / Judge Reliability。它在当前 vault map 里是一个横切基础设施主题：RAG、Agent、Deep Research Agents、长上下文系统最后都要被评测，而评测一旦依赖 LLM judge，就必须先问 judge 本身是否稳定、可校准、可追踪。

本周位置

这是 adjacent topic。RAG 是当前锚点；Evaluation 不是直接改进检索或生成，但它决定我们如何判断 RAG 输出是否有证据、引用是否真实、失败来自检索还是生成，以及自动 judge 能不能替代人工阅读。

和 RAG 的关系

RAG 的核心承诺是“答案有证据支撑”。但如果 judge 只看回答是否顺滑，或者引用 URL、证据覆盖、上下文一致性没有被独立验证，RAG 会变成带链接的幻觉系统。今天的 Evaluation 线直接服务 RAG：它把“看起来引用了资料”拆成 citation validity（引用是否真实）、evidence faithfulness（证据是否支持结论）、per-instance reliability（单个样本上的 judge 判断是否可信）几层。

本主题你现在需要知道的 3 件事

1. LLM-as-a-judge 已经不能只看平均相关性。 新问题是：同一个 judge 在某个具体样本上是否可靠？它的偏好是否满足传递性？它是否只是被表面格式、上下文暗示或自己不知道的问题带偏？

2. citation validity 正在从 Deep Research / RAG 的附属检查变成 judge reliability 主线。 GhostCite 和 4 月新论文都在说明：引用不是可有可无的格式项，而是研究型系统可信度的底座。

3. 多模态和高风险领域正在暴露新型 judge bias。 当 judge 需要同时整合图片、问题、回答、领域参考答案时，偏差不再只是“偏长答案”或“偏某个位置”，而是 modality neglect（忽略某个模态证据）、reference-free grading failure（没有参考答案时无法判断正确性）和 judge disposition（严格 / 宽松 judge 的反馈性格）共同作用。

What Changed

1. 评测前沿从“judge 是否和人类一致”推进到“每个样本上 judge 是否可置信”。 Diagnosing LLM Judge Reliability 用 conformal prediction sets（保覆盖率的预测集合）给 Likert 分数输出不确定性宽度，并用 transitivity violations（偏好传递性违背）发现平均低违背率会掩盖大量文档级不一致。

2. 引用可靠性开始可大规模测量，也可被工具化修正。 Detecting and Correcting Reference Hallucinations 在 DRBench 和 ExpertQA 上测 citation URL validity，并用 Wayback Machine 区分 hallucinated URL 和 link rot；它还显示接入 urlhealth 后，部分 agentic self-correction 能把 non-resolving citation URL 降到 1% 以下。

3. 没有人工 grounding 的 judge 有硬边界。 No Free Labels 的 2026-04-01 修订版强调：在商业 / 金融这类 correctness-critical 领域，judge 如果自己答不对问题，也很难可靠判断别人的答案；专家参考答案比单纯换更强 judge 更关键。

4. 多模态 judge 开始出现专门的 bias benchmark。 MM-JudgeBias 把 Query、Image、Response 分别扰动，测 Bias-Deviation 和 Bias-Conformity，说明 MLLM-as-a-Judge 会在证据缺失、图文不匹配和无关扰动下系统性失稳。

5. 社群信号偏弱，不能主导今天排序。 本机 last30days 只跑到 Reddit-only，结果多为泛 AI 讨论和模型榜单噪声；唯一可保留的背景信号是社区对 benchmark 半衰期、PPL/KLD 等代理指标失效的怀疑在增加。今天应按论文证据排序，而不是按社媒热度排序。

Read Now

1. Diagnosing LLM Judge Reliability: Conformal Prediction Sets and Transitivity Violations

  • why now：这是今天最贴近 Judge Reliability 主线的新论文，直接把可靠性落到 per-instance 层，而不是只报告整体相关性。
  • method / benchmark / mechanism point：在 SummEval 上做两类诊断：一是偏好图里的 3-cycle，用来测传递性违背；二是 split conformal prediction sets，用集合宽度表示该样本的 judge 不确定性。
  • how to read it first：先看 transitivity analysis 的定义，再看 prediction set width 为什么能跨 judge 一致地反映文档难度；最后看哪些评价维度最不可靠。
  • source link：https://arxiv.org/abs/2604.15302

2. Detecting and Correcting Reference Hallucinations in Commercial LLMs and Deep Research Agents

  • why now：它把 Deep Research / RAG 最要命的“引用 URL 是否真实”做成可测量、可纠正的问题，正好补本地 GhostCite 队列。
  • method / benchmark / mechanism point：在 DRBench 和 ExpertQA 上检查 221k+ URL，用 Wayback Machine 区分从未存在的 hallucinated URL 与真实但失效的 link rot，并发布 urlhealth 工具。
  • how to read it first：先读 failure taxonomy，再看 deep research agents 为什么比 search-augmented LLM 生成更多引用但也可能更高幻觉率；最后看 self-correction 对不同模型为何差异巨大。
  • source link：https://arxiv.org/abs/2604.03173

3. No Free Labels: Limitations of LLM-as-a-Judge Without Human Grounding

  • why now：它 2026-04-01 更新后非常适合今天补读，因为它给出一个明确边界：judge 缺少领域知识时，reference-free evaluation 会失效。
  • method / benchmark / mechanism point：BFF-Bench 使用金融专业人士写的问题和长回答，并让专家标注 1,200 个 response correctness；核心发现是专家参考答案能显著缓解 judge 的知识边界。
  • how to read it first：先读“judge 能否回答问题”和“judge 能否判断答案”的耦合关系，再看 expert-written reference 如何改变弱 judge 与强 judge 的相对表现。
  • source link：https://arxiv.org/abs/2503.05061

4. MM-JudgeBias: A Benchmark for Evaluating Compositional Biases in MLLM-as-a-Judge

  • why now：Evaluation 主题不能只停留在文本 judge；多模态 RAG 和多模态 agent 后续都会需要 MLLM-as-a-Judge。
  • method / benchmark / mechanism point：MM-JudgeBias 从 29 个源 benchmark 构造 1,800+ 多模态样本，系统扰动 Query、Image、Response，并用 Bias-Deviation / Bias-Conformity 分别测敏感性和稳定性。
  • how to read it first：先看九类 compositional bias 的定义，再看哪些模型在图像证据缺失或文本干扰下仍给出过度自信判断。
  • source link：https://arxiv.org/abs/2604.18164

Watchlist

  • GhostCite: A Large-Scale Analysis of Citation Validity in the Age of Large Language Models

本地队列高优先级。它更像 citation validity 的宏观底座：13 个模型、40 个领域、AI/ML 与安全会议 56,381 篇论文。今天不放 Read Now 是因为 4 月 URL-level correction 论文更直接连接 deep research agents。

  • Confidence Estimation for LLMs in Multi-turn Interactions

本地队列高优先级。它关注多轮上下文里的 confidence calibration 和 monotonicity，适合与 agent / RAG 多轮评测一起读；今天可暂后，因为主线更偏 judge 可靠性诊断。

  • Are We on the Right Way to Assessing LLM-as-a-Judge?

值得跟踪。Sage 用 local self-consistency 和 global logical consistency 不依赖人工标签地评估 judge，和今天的 transitivity diagnostics 高度同向。

  • Exploiting LLM-as-a-Judge Disposition on Free Text Legal QA via Prompt Optimization

适合放在法律 / 高风险评测 watchlist。它显示 lenient judge 和 strict judge 在 prompt optimization 里会诱导不同泛化行为，说明 judge 的“性格”会污染下游优化。

  • When Judgment Becomes Noise

这是 benchmark validity 的结构性提醒。它用 schematic adherence 和 psychometric validity 指出 Arena-Hard Auto 这类 LLM-judged benchmark 可能把真实不确定性折叠成高置信排名。
