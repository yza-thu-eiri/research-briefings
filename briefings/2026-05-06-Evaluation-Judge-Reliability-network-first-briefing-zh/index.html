今日主题：Evaluation / Judge Reliability

今天是周三，轮转主题是 [[Evaluation]] / [[Judge Reliability]]。在当前 vault map 里，它是 RAG、Agent、Deep Research Agents 的共同测量层：如果 judge 本身不稳定，任何“检索更好”“引用更准”“agent 报告更强”的结论都会变成不可靠的二手信号。

本地主题笔记已经把 Evaluation 从 benchmark 总分推进到几个更硬的问题：逐样本可靠性、human grounding、verifier-based evaluation、引用有效性和可追溯评估。今天的网络扫描显示，前沿又往前走了一步：不是只证明 LLM judge 有偏差，而是开始系统测量 judge 结论对风格、prompt 改写、安全配置和自我偏好的敏感性。

本周位置

Adjacent topic。

RAG 是本周锚点；Evaluation / Judge Reliability 不直接改进检索器，但决定我们能否可信地判断 RAG 输出是否有证据、引用是否真实、答案是否只是顺滑地幻觉。

和 RAG 的关系

RAG 的核心承诺是“答案由外部证据支撑”。但这件事最后常常交给 judge 或 evaluator 判断：引用是否有效、证据是否支持断言、检索是否覆盖关键来源、答案是否忠实于上下文。若 judge 对 prompt wording、候选顺序、答案风格或自身生成内容有系统偏差，RAG 评测就会把测量噪声误读成系统进步。

本主题你现在需要知道的 3 件事

1. Judge reliability 已经从平均分问题变成协议敏感性问题。 现在要问的不只是“这个 judge 和人类相关性多高”，还要问“换一种等价 prompt、换一种安全 judge 配置、换一组风格不同但质量相当的答案，结论还稳不稳”。

2. Style bias 正在从次要偏差变成主偏差。 最新系统评估显示，style bias 可能远大于 position bias；这意味着 judge 可能不是偏第一个答案，而是偏某种表达风格、组织方式或语气。

3. 没有 grounding 的 judge 标签仍然不能当免费真值。 本地队列中的 No Free Labels 仍是底线论文：正确性任务里，如果没有可靠 reference、人类验证或程序化 verifier，LLM judge 输出本身就需要被审计。

前沿变化观察

1. Debiasing 开始被当成 pipeline 设计，而不是 prompt 小修。

Judging the Judges 同时比较 9 种 debiasing 策略、5 个 judge 模型、3 个 benchmark 和 4 类 bias。最重要的变化是：研究对象从单个 judge 变成“judge pipeline 如何配置才更稳”。

2. Prompt sensitivity 被定义成 judge 自身的基准。

JudgeSense 用语义等价的 prompt paraphrase 测 judge verdict 是否保持一致，并提出 Judge Sensitivity Score。这个方向和本地 Prompt Sensitivity 主题直接相连：prompt 不只是模型输入噪声，也是评测结论的方差来源。

3. Safety benchmark 的测量方差被拉到台前。

How Sensitive Are Safety Benchmarks to Judge Configuration Choices? 显示，在 HarmBench 类安全评测中，仅调整 judge prompt 的结构和措辞，就能显著改变 harmful-response rate。安全榜单因此不能只报告模型分数，还需要报告 judge configuration。

4. Self-preference bias 不一定随模型能力提升而下降。

Quantifying and Mitigating Self-Preference Bias 的核心提醒是：强模型不自动等于低自偏。用模型评自己或同源模型的输出时，评测系统可能把 evaluator identity 当成隐藏变量。

5. Groundedness 正在从单一分数转向 typed evidence + recovery。

GSAR 把 claim 分成 grounded、ungrounded、contradicted、complementary，并把不同证据类型赋予不同权重，再决定 proceed / regenerate / replan。这比单一 groundedness 分数更接近 RAG / Agent 的实际修复流程。

Read Now

1. Judging the Judges: A Systematic Evaluation of Bias Mitigation Strategies in LLM-as-a-Judge Pipelines

  • why now： 这是今天最值得先读的增量论文。它不再停留在“judge 有偏差”，而是比较不同 debiasing 策略在真实 pipeline 里是否有效。
  • method / benchmark / mechanism point： 9 种 debiasing 策略、5 个 judge 模型、3 个 benchmark、4 类 bias；关键发现是 style bias 在所有模型上都很强，而 position bias 反而较小。
  • how to read it first： 先读 bias taxonomy 和控制数据构造，再读“style bias 为什么主导”。最后看 combined budget strategy 对不同 judge 的效果是否可迁移。
  • source link： https://arxiv.org/abs/2604.23178

2. JudgeSense: A Benchmark for Prompt Sensitivity in LLM-as-a-Judge Systems

  • why now： 它把 judge 的 prompt sensitivity 做成独立评测对象，正好补本地 Prompt Sensitivity 与 Judge Reliability 的交叉。
  • method / benchmark / mechanism point： 494 组验证过的 prompt paraphrase pair，评测 9 个 judge；Judge Sensitivity Score 表示等价改写下 verdict 保持一致的比例。
  • how to read it first： 先看 JSS 定义和 paraphrase 构造，再看 factuality prompt artifact 与 pairwise always-A 行为。阅读目标是识别哪些评测协议本身会制造不稳定。
  • source link： https://arxiv.org/abs/2604.23478

3. How Sensitive Are Safety Benchmarks to Judge Configuration Choices?

  • why now： 安全评测通常最需要可信，但这篇说明 judge prompt wording 本身会显著改变有害率测量和模型排名。
  • method / benchmark / mechanism point： 2 x 2 x 3 factorial design，12 种 judge prompt variant，28,812 条 HarmBench 判断；在固定 judge model 下，prompt wording 可让 measured harmful-response rate 产生大幅波动。
  • how to read it first： 先读实验设计，不要先看排名；重点看 evaluation structure 和 instruction framing 两个轴如何改变测量结果。
  • source link： https://arxiv.org/abs/2604.24074

4. No Free Labels: Limitations of LLM-as-a-Judge Without Human Grounding

  • why now： 虽然它不是今天最新论文，但仍是本地队列里最重要的底线论文：没有 grounding 的 judge label 不应被默认当成正确性真值。
  • method / benchmark / mechanism point： BFF-Bench 与 VERDICTS 收集专家正确性标注，比较无 reference、合成 reference、人类/专家 reference 对 judge 可靠性的影响。
  • how to read it first： 先读“judge 自己能否答对问题”和“judge 能否判断候选答案”的耦合关系；再读错误 reference 如何污染裁判。
  • source link： https://arxiv.org/abs/2503.05061

5. Diagnosing LLM Judge Reliability: Conformal Prediction Sets and Transitivity Violations

  • why now： 它是本地 Judge Reliability 当前最好的逐样本诊断入口，和今天的新 prompt/bias 论文形成互补。
  • method / benchmark / mechanism point： 用有向三元环检测成对偏好中的传递性违背，用 split conformal prediction set 宽度表达单条评分的不确定性。
  • how to read it first： 先看 transitivity violations 和 conformal set width 两个诊断信号；再看不同评价维度的可靠性差异。阅读时保留一个疑问：重复采样预算不对称会不会混入协议噪声。
  • source link： https://arxiv.org/abs/2604.15302

Watchlist

  • Quantifying and Mitigating Self-Preference Bias of LLM Judges

技术上重要，但今天可先 watch。它提出自动构造等质量 response pair 来分离 discriminability 和 self-preference bias，并用多维结构化评估降低自偏。后续若研究“模型评自己”或同源 judge，应补读。

  • GSAR: Typed Grounding for Hallucination Detection and Recovery in Multi-Agent LLMs

值得和 RAG/Agent 线一起跟。它把 evidence 类型和 recovery action 连起来，不只是输出 groundedness scalar；适合后续做 RAG 证据诊断和 agent replan 机制时读。

  • LLM-as-a-Judge for Human-AI Co-Creation: A Reliability-Aware Evaluation Framework for Coding

放在 coding agent / human-AI collaboration watchlist。它用 rubric-driven schema、校验修复、trajectory-level signals 和概率可靠性指标评估 co-creation，但 inter-judge consistency 仍然偏低。

  • GhostCite: A Large-Scale Analysis of Citation Validity in the Age of Large Language Models

本地高优先级队列项。今天不放 Read Now 是因为近期增量更集中在 judge 协议敏感性；但 GhostCite 仍是引用有效性基础入口，应和 BibAgent、Deep Research 引用核验一起读。

  • Confidence Estimation for LLMs in Multi-turn Interactions

本地高优先级队列项。它把置信度估计放进多轮信息累积场景，适合连接 Agent / Dialogue evaluator；今天先 deferred，因为主线更偏 judge pipeline 的方差来源。
