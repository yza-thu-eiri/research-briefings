今日主题：Agent / Deep Research Agents

今天是周二，轮转到 [[Agent]] / [[Deep Research Agents]]。这次不是重复 05-12 那批 benchmark、verification-centric design 和 reasoning-aware retrieval，而是看最近增量：deep research agent 的瓶颈正在从“能不能写长报告”继续下沉到三个更硬的接口：引用 URL 是否真实存在、综述式研究能不能找全并组织文献、以及检索器能力是否真的被 agent 的查询行为用起来。

What I Learned

  • 引用有效性变成了可量化的主问题。 Reference hallucination 指模型给出的引用 URL 看起来像证据，但链接可能从未存在或已经不可解析。新工作把这个问题放到 commercial LLMs 和 deep research agents 上系统测量，并给出 urlhealth 这类可接入 agent 自纠错回路的工具。
  • 综述生成不再只看写得像不像。 TaxoBench 把 deep research agent 拉到“能否找回专家综述里的关键论文，并按专家式 taxonomy 组织”的层面。这个口径对本地 vault 很重要，因为它更接近“能不能帮人建立知识结构”，而不只是“能不能生成一篇报告”。
  • 检索问题仍然没有被 LLM-based retriever 自动解决。 SAGE 的反直觉信号是：如果 agent 发出的仍是 keyword-oriented subqueries，更复杂的 LLM retriever 不一定比 BM25 好。这说明 deep research 的 retrieval bottleneck 很可能在 agent query formulation 和 corpus representation 的接口处。
  • 社区信号只能当弱背景。 本轮 last30days 覆盖主要是 Reddit，且噪声较高；能保留的只是一个方向性判断：大家仍在讨论 scaffold、harness 和 tool-calling discipline 对 agent 表现的影响，而不是只看 base model。

Key Patterns

1. 从 citation faithfulness 走向 URL-level audit。 之前的重点多是“引用是否支持断言”；现在进一步细化到 URL 本身是否可解析、是否曾经存在、是 link rot 还是 fabricated URL。

2. 从 report-quality benchmark 走向 retrieval-plus-organization benchmark。 研究型 agent 如果连专家会引用的核心论文都找不全，或者拿到论文后组织不出合理层级，那么再流畅的报告也只是表层能力。

3. 从更强 retriever 走向 agent-retriever interface。 SAGE 和 05-12 已读的 AgentIR 形成一个有用张力：一边说 reasoning trace 应该进入检索，一边说明当前 agent 真实发出的查询常常仍然太像关键词。

4. 从事后检查走向可插拔的修复工具链。 urlhealth 和 citecheck 都说明 citation repair 正在变成 agentic scholarly workflow 的基础设施，而不是人工审稿后的补丁。

Read Now

1. Detecting and Correcting Reference Hallucinations in Commercial LLMs and Deep Research Agents

  • why now: 这是今天最值得读的一篇。它把 deep research agents 的引用问题从泛泛的“会幻觉”拆成可测的 URL validity：从未存在的 fabricated URL、真实但失效的 link rot、以及整体 non-resolving citation rate。
  • what changed: 论文在 DRBench 和 ExpertQA 上做大规模统计，指出 deep research agents 往往生成更多引用，但也可能有更高的 URL 幻觉率；加入 urlhealth 后，部分模型能把 non-resolving citation URLs 降到 1% 以下。
  • read it for: 如果你想把 Deep Research Agents 这条线接回 citation validity / evidence faithfulness，这篇是最直接的新锚点。
  • how to start: 先看 failure taxonomy，再看 urlhealth 的 stale-vs-hallucinated 判断逻辑，最后看 self-correction 实验里不同模型工具使用能力的差异。

2. Can Deep Research Agents Retrieve and Organize? Evaluating the Synthesis Gap with Expert Taxonomies

  • why now: 这篇 v3 在 2026-05-09 更新，正好补上 deep research agent 的另一个盲区：不是只有 citation correctness，综述式研究还要求找全关键论文并组织成知识结构。
  • what changed: TaxoBench 用 72 篇高引用 CS survey 的专家 taxonomy 和 3,815 篇被分类论文做 ground truth，同时评估 retrieval 和 organization。结果显示，最好的 agent 也只找回约 20.92% 的专家引用论文。
  • read it for: 如果你关心本地 vault 的“研究记忆编译器”方向，这篇比一般 benchmark 更贴近核心问题：机器能不能把读到的文献组织成可成长的知识结构。
  • how to start: 先读 Deep Research mode vs Bottom-Up mode 的区分，再看 capability-based 和 alignment-based findings。

3. SAGE: Benchmarking and Improving Retrieval for Deep Research Agents

  • why now: 它把 deep research agent 的检索子问题单独拿出来做 scientific literature retrieval benchmark，和本地 RAG、DeepSearchQA、AgentIR 都有交叉。
  • what changed: 论文发现六个 deep research agents 都在 reasoning-intensive retrieval 上吃力，而且 BM25 反而能显著超过若干 LLM-based retrievers，原因之一是 agent 生成的子查询仍偏 keyword-oriented。
  • read it for: 用来校准一个判断：deep research agent 的瓶颈不一定是没有更强的 retriever，而是 planner、query formulation 和 corpus metadata 没有对齐。
  • how to start: 先看 1,200 queries / 200,000 paper corpus 的任务设定，再看 BM25 vs ReasonIR / gte-Qwen2-7B-instruct 的对比。

Watchlist

  • citecheck: An MCP Server for Automated Bibliographic Verification and Repair in Scholarly Manuscripts

这不是 deep research benchmark，但它是很好的基础设施信号。它把 .bib、.tex、.md、.txt、.docx 中的参考文献抽取出来，对 PubMed、Crossref、arXiv、Semantic Scholar 做验证，并通过 MCP 暴露给 agentic scholarly editing workflow。

  • Agent scaffold / harness 社区讨论

本轮 last30days 的覆盖不足以支撑强结论，但弱信号仍然一致：agent 表现越来越被 scaffold、tool calling、harness 和安全网影响。对本主题来说，它提醒我们看 deep research agent 时不能只比较底层模型名。

Signal Check

本轮 signal line 是 reddit-only，覆盖质量偏低，不应主导排序。可保留的背景判断只有一个：外部讨论正在继续把 agent 能力归因到 scaffold 和 tool discipline，而 paper side 则更精确地把这些问题落到 citation checking、retrieval query behavior 和 taxonomy organization 上。

Recommended Next Reads

1. 先读 Reference Hallucinations in Commercial LLMs and Deep Research Agents，因为它最直接更新 citation validity 主线。

2. 再读 TaxoBench，把 deep research agent 从“报告生成器”重新拉回“文献组织者”的评估口径。

3. 最后读 SAGE，用它校准 AgentIR 之后的检索路线：reasoning-aware retrieval 要成立，必须先让 agent 输出的检索状态足够可用。
